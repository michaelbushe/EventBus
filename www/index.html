About the EventBus

The Event Bus is a Swing-oriented single-process publish/subscribe event routing library. The EventBus is fully-functional, with very good API documentation and test coverage (80+%). It has been deployed in production.

The Event Bus is easy to use and yet very powerful. It requires no setup and the API is small and clear. The EventBus solves many problems, particularly with Swing applications.

The slides from the EventBus presentation at JavaOne 2006 demonstrate many ways to use the EventBus. The code from the talk is not yet available, but will be soon.

Though not Swing-specific, the EventBus has Swing support and the Event Bus hopes to make Swing applications more componentized, modular and testable. The EventBus has its conceptual roots in the old Info Bus from JDK 1.1 days, but it is much simpler and smaller. The idea is similar to a computer system bus - components plug into it and listen for information passed on it. This is different from the Swing listener paradigm where listeners attach to and listen on specific components. The Event Bus is related to messaging systems like JMS whereby processes publish messages on a topic and other systems can listen on the topic. The JMS implementation takes care of efficiently routing the messages. The major advantage to JMS and the EventBus is that it introduces a decoupling layer. Components only have to know about the bus, not each subscriber and publisher that they may be interested in.

Components can subscribe to events passed on the Event Bus by Event Class or by a topic name (String). Subscriptions can be done weakly, avoiding memory leaks.

Though code complete, this project is new, so the project website is not yet complete.

Examples

More examples on the way, but here's a quick one on making a status bar:

StatusEvent.java 

public StatusEvent { 
   Sting statusText;
   
   public StatusEvent(String statusText) { 
      this.statusText = statusText;
   }
   public String getStatusText() {
      return statusText;
   }
}

StatusBar.java 

public StatusBar extends JLabel implements EventSubscriber {
   public StatusBar() {
      EventBus.subscribe(StatusEvent.class, this);
   }
   public void onEvent(Object evt) {
      StatusEvent statusEvent = (StatusEvent)evt;
	  this.setText(statusEvent.getStatusText();
   }
   }

}

AnySwingControll.java 

...   
EventBus.publish(new StatusEvent("All data loaded."));
 

With Annotations, it's a bit simpler:

StatusEvent.java 

public StatusEvent { 
   Sting statusText;
   
   public StatusEvent(String statusText) { 
      this.statusText = statusText;
   }
   public String getStatusText() {
      return statusText;
   }
}

StatusBar.java 

public StatusBar extends JLabel implements EventSubscriber {
   public StatusBar() {
      AnnotationProcessor.process(this);
   }
   @EventSubscriber(eventClass=StatusEvent.class)
   public void onEvent(StatusEvent statusEvent) {
	  this.setText(statusEvent.getStatusText();
   }
   }

}

Future

Future goals include:

  • Create tutorials
    • @Publish annotation for Swing fields to autoconvert addXXXlisteners boilerplate to EventBus events.
    • Simple usage Swing component usages for such components as a progress bar.
    • moving information from non-Swing threads to Swing components via events.
    • mvc models through events.
    • form validation solutions through events.
    • integrating services-oriented Swing with events.
  • Get eyes on it and feedback!

Feedback

Though there is not a lot of traffic, this is an active project. Feedback is welcome and is turned around very quickly. The EventBus is tiny, so changes are easy to make and we are always looking for feedback and improvement. So please, send an email to tell us how you are using it, or become a contributor. We are actively looking for improvements in the core in Annotations and Generics, and we are also looking to add supplemental libraries that encapsulate usages of the EventBus (like an EventBus-aware StatusBar, for example).
